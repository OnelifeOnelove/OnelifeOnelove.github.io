相关概念

  • 镜像（image）
    一个可以独立运行的特殊文件系统，它是静态的，我们可以从官方仓库拉取一些基础镜像，然后把我们的应用放入其中，打造一个新的包含我们饮用的镜像。

  • 容器（container）
    一个镜像运行起来后就是一个镜像，这就跟Java中类跟对象的关系一般，镜像有一个，但是我可以多起几个容器，即：容器是运行中的镜像。

  • 仓库（repository）
    Docker Registry是一个集中管理镜像服务的地方，它其中可以有多个仓库，每个仓库中存储的都是镜像，默认以仓库:标签的方式定位一个镜像。在docker中，要删除一个镜像，就要先关闭当前镜像开启的容器，然后用rmi id删除镜像，rm是删除容器，如果一个镜像挂载到多个仓库，那么可以rmi 仓库:标签的方式进行删除。
